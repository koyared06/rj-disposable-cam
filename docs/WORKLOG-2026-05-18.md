# Camera Module Worklog - 2026-05-18

Date: May 18, 2026
Project: RJ Disposable Camera

## Summary
Today we focused on camera stability, moderation workflow, gallery behavior, and event branding consistency across guest and admin flows.

## Completed Changes

1. Dynamic event branding in camera and print outputs
- Added admin-configurable camera branding fields and connected them to:
  - Guest camera landing page
  - QR center label
  - Print QR layouts (QR card, A4/table cards, entrance poster)
- Updated typography usage for event title/hashtag/tagline in relevant camera/QR views.

2. Guest upload reliability (Google Sheets/Drive)
- Improved quota/rate-limit handling in upload API.
- Added retry with exponential backoff and jitter in server upload path.
- Returned retry metadata (`429`, `Retry-After`) to support client-side recovery.
- Added queue-oriented auto-retry flow in guest camera upload pipeline.
- Kept failed items in queue with retry scheduling instead of hard-failing immediately.

3. Image processing before upload
- Kept/extended pre-upload compression + watermark pipeline for guest shots.
- Multi-quality fallback to reduce file size while preserving acceptable quality.

4. Guest name validation UX
- Enforced guest name minimum length in UI flow.
- Added inline note/error messaging and disabled submit/related actions when invalid.
- Fixed landing "I Understand" behavior so users can continue to camera flow and complete name step there.

5. Gallery lock and reveal behavior
- Fixed gallery lock timing to use Manila wall-clock interpretation consistently.
- Prevented accidental admin bypass in guest links by requiring explicit `adminView=1` for admin-view behavior in `/cam`.

6. Gallery visibility and POV completeness
- Updated guest gallery listing behavior so approved non-own photos are included in list/POV set.
- Result: `Choose a POV` now has complete approved uploader options.
- Blur/lock behavior remains on guest UI before reveal time.

7. Camera Upload Moderation bulk actions
- Added selection model in admin moderation table:
  - Per-row checkbox
  - Select/Unselect current page
  - Select filtered
  - Clear selection
- Added bulk actions:
  - Bulk Approve
  - Bulk Hide
  - Bulk Reject
  - Bulk Delete (with strict `DELETE` confirmation)
- Preserved failed IDs selected for quick retry.

8. Thumbnail/broken tile stability
- Added retry/backoff to `/api/camera/file` Drive download path for transient quota/network failures.
- Improved cache-control for image responses to reduce repeated fetch pressure.
